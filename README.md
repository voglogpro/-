# БибиЗадачи — новый бот заданий и бибибонусов

Отдельный бот и мини-приложение (НЕ трекер смен). Здесь пользователи из
группы подают заявку, берут задания на карте, выполняют их и получают
бибибонусы на бесплатные поездки. Дизайн и концепт взяты из рабочего
трекера смен, механика — новая.

## Файлы

| Файл | Что это |
|---|---|
| `main.py` | Код бота — точка входа (это запускает хостинг) |
| `index.html` | Страница мини-приложения (отдаётся по адресу сайта) |
| `requirements.txt` | Зависимости |
| `amvera.yml` | Конфиг для Amvera (указывает, что запускать) |

Кладутся в корень **отдельного** репозитория, рядом.

## Почему это точно не столкнётся с трекером смен

Файлы называются так же, как у трекера, но конфликта нет:

- Новый бот лежит в **отдельном репозитории** — одинаковые имена файлов в
  разных репозиториях не пересекаются.
- Отдельная **база**: `bibitasks.db` (у трекера — `bibibike_work.db`).
- Отдельный **токен** и отдельный бот в BotFather.

Главное правило: заливать в **отдельный** репозиторий, не в тот, где трекер
смен. Данные трекера в полной безопасности.

## Переменные окружения (на хостинге)

Обязательные:
- `BOT_TOKEN` — токен НОВОГО бота от @BotFather (отдельный бот, не трекер смен)
- `BOT_USERNAME` — username бота без @ (для кнопок и реферальных ссылок)
- `ADMIN_IDS` — Telegram ID ответственных через запятую, например `111,222`.
  Эти люди сразу получают роль «ответственный»: одобряют заявки, создают
  задания, подтверждают выполнение и начисляют бонусы.

Полезные:
- `DATA_DIR` — папка постоянного хранилища для базы (на Amvera `/data`).
  Без неё база теряется при пересборке.
- `WEBAPP_SHORTNAME` — короткое имя Direct Link (по умолчанию `app`)
- `COMMUNITY_CHAT_ID` — ID группы сообщества (задел на будущее)

Свой Telegram ID можно узнать у @userinfobot.

## Настройка в BotFather

1. Создай нового бота → получи токен → впиши в `BOT_TOKEN`.
2. Mini Apps → Main App / Menu Button / Direct Link → укажи домен хостинга.
3. Direct Link короткое имя — то же, что в `WEBAPP_SHORTNAME`.

## Как это работает

**Обычный пользователь:**
1. Открывает бота из группы (тема «Работа») → видит приветствие.
2. Жмёт «Открыть задания» → подаёт заявку (имя + телефон).
3. Ждёт одобрения (придёт уведомление в чат).
4. После одобрения — берёт задания на карте, выполняет, жмёт «Готово».
5. Ответственный подтверждает → бибибонусы падают в кошелёк.
6. Тратит бонусы на поездки (пока — через промокод от ответственного).

**Ответственный (из ADMIN_IDS):**
- Вкладка «Модерация»: одобряет заявки, создаёт задания, подтверждает
  выполнение, видит команду и балансы, начисляет/списывает бонусы вручную.

**Уровни доверия:** Новичок → Проверенный (10 заданий) → Амбассадор (40).
Растут за подтверждённые задания.

**Рефералы:** кнопка «Поделиться ссылкой» в профиле. Друг регистрируется по
ней и после одобрения приглашающему падает +50 бонусов.

## Что готово и проверено

- Полный цикл: заявка → одобрение → задание → взятие → выполнение →
  подтверждение → бонусы (протестировано end-to-end).
- Защита: без подписи Telegram — отказ, поддельная подпись — отказ,
  неодобренные не видят задания и модерацию.
- Античит-основа: задание нельзя взять дважды, все бонусы логируются с
  причиной.
- Веб-сервер стартует первым и не роняет бота.

## Что дальше (по плану платформы)

- Трата бонусов на поездки автоматически — когда появится интеграция с
  системой аренды Бибибайк (пока полуручно через промокод).
- Гео-проверка выполнения и фото — для дорогих заданий (Фаза 4 плана).
- Кнопка «Хочу помогать» для полностью посторонних — расширение за пределы
  сотрудников (Фаза 5). Сейчас заявку может подать любой, кто открыл бота.

## Важно

Это самостоятельный бот с отдельной базой `bibitasks.db`. Он никак не
пересекается с трекером смен — можно запускать параллельно, на разных
токенах. Данные трекера в безопасности.
